配置环境

安装Node（必须）
作用：用来生成静态页面的
安装Git（必须）
作用：把本地的hexo内容提交到github上去
申请GitHub（必须）
作用：是用来做博客的远程创库、域名、服务器之类的，怎么与本地hexo建立连接等下讲。

安装Hexo

Node和Git都安装好后，首先创建一个文件夹，如blog，用户存放hexo的配置文件，然后进入blog里安装Hexo。

配置Github

建立Repository

github主页右上角有个‘加号’，选择New repository

建立与你用户名对应的仓库，仓库名必须为【your_user_name.github.io】
然后直接create repository

生成SSH Keys

我们如何让本地git项目与远程的github建立联系？这时候就要用到SSH Keys
首先设置你的用户名密码：（在git bash中输入）

生成密钥：

然后按下三个enter，按照上面出现的位置____/.ssh/id_rsa.pub，找到id_rsa.pub打开并复制密钥

添加SSH Key到Github

登陆Github,点击右侧用户按钮，选择Settings
点击 Add SSH key 按钮，将复制的密钥粘贴到 Key 栏

测试能不能链接成功：

执行结果：

现在你已经可以通过SSH链接到Github了

现在我们需要_config.yml文件，来建立关联
打开_config.yml翻到最下面，改成下面这样（注意冒号后一定要加空格！！！）

deploy:
     type: git
     repo: https://github.com/你的GitHub名/你的GitHub名.github.io.git
     branch: master

更多_config.yml相关配置 👉https://hexo.io/zh-cn/docs/configuration.html

部署

然后执行命令：
​ npm install hexo-deployer-git –save

执行配置命令：
​ hexo deploy（hexo d）

然后再浏览器中输入 http://用户名.github.io/ 就行了

每次更新部署的步骤，可按以下步骤来进行：

hexo generate
hexo deploy

Hexo中所有的文章都用markdown语言编写（墙裂推荐用Typora编辑器）

一些常用命令

命令 作用
hexo new “文章名称” 新建文章
hexo generate 生成静态页面至public目录
hexo server 预览访问（默认端口4000，’ctrl + c’关闭server）
hexo deploy 将.deploy目录部署到GitHub
hexo version 查看Hexo的版本
